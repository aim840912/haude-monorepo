## 1. 專案概述

### Q1.1：這個專案是做什麼的？主要解決什麼問題？

**答案**：

豪德製茶所是一個**全端電商平台**，專為台灣在地茶葉品牌設計，結合兩大核心功能：

1. **線上茶葉銷售**
   - 完整的購物車與結帳流程
   - 會員等級制度（普通、銅卡、銀卡、金卡）
   - 折扣碼與促銷管理
   - 產品評論系統

2. **農村旅遊體驗預訂**
   - 茶園導覽、採茶體驗、工作坊
   - 線上預約與人數管理
   - 市集日程公告

**解決的痛點**：
- 傳統茶農缺乏數位銷售管道
- 農村旅遊體驗難以線上預約
- 會員經營缺乏系統化工具

### Q1.3：這個專案包含哪些應用程式？

**答案**：

採用 **Monorepo 架構**，包含三個獨立應用：

```
┌─────────────────────────────────────────────────────────────┐
│                    haude-monorepo                           │
├─────────────────────────────────────────────────────────────┤
│  apps/web      │ 用戶端網站 (Next.js 15)  │ Port 5173      │
│  apps/admin    │ 管理後台 (Vite + React)  │ Port 5174      │
│  apps/api      │ REST API (NestJS 11)     │ Port 3001      │
├─────────────────────────────────────────────────────────────┤
│  packages/types │ 共用 TypeScript 型別                      │
└─────────────────────────────────────────────────────────────┘
```

**優點**：
- 共用型別確保前後端一致性
- 統一指令管理 (`pnpm dev` 一鍵啟動)
- 智慧建置快取加速開發

### Q2.2：前後端如何溝通？API 設計是什麼風格？

**答案**：

採用 **RESTful API** 設計，所有端點都有 `/api/v1` 前綴：

```
認證相關：
POST   /api/v1/auth/register    # 註冊
POST   /api/v1/auth/login       # 登入
GET    /api/v1/auth/me          # 取得當前用戶

產品相關：
GET    /api/v1/products         # 產品列表
GET    /api/v1/products/:id     # 產品詳情

訂單相關：
GET    /api/v1/orders           # 訂單列表
POST   /api/v1/orders           # 建立訂單
PATCH  /api/v1/orders/:id/cancel # 取消訂單

管理端點：
POST   /api/v1/admin/products   # 建立產品（需管理員權限）
```

**API 文件**：整合 Swagger，開發環境可於 `/docs` 查看完整 API 文件。

## 3. 功能特色

### Q3.1：會員等級系統如何運作？

**答案**：

**四級會員制度**：

| 等級 | 累積消費門檻 | 專屬折扣 | 積分倍率 | 免運費 |
|------|-------------|----------|----------|--------|
| 普通會員 | - | 無 | 1x | 否 |
| 銅卡會員 | NT$3,000 | 95折 | 1.2x | 否 |
| 銀卡會員 | NT$10,000 | 92折 | 1.5x | 是 |
| 金卡會員 | NT$30,000 | 88折 | 2x | 是 |

**特色功能**：
- **自動升級**：消費累積達標自動升級
- **生日獎勵**：生日當月雙倍積分
- **積分兌換**：可用於折抵訂單金額
- **歷史記錄**：完整的等級變更與積分交易記錄

---

### Q3.2：支援哪些付款方式？

**答案**：

整合 **ECPay（綠界科技）**，支援台灣主流付款方式：

| 付款方式 | 說明 |
|----------|------|
| **信用卡** | 一次付清、分期付款 |
| **ATM 轉帳** | 虛擬帳號，繳費期限內完成 |
| **超商代碼** | 7-11、全家、萊爾富、OK |

**付款流程**：
1. 用戶下單 → 系統建立訂單
2. 導向 ECPay 付款頁面
3. 付款完成 → ECPay 回調通知
4. 系統更新訂單狀態 → 發送確認通知

**安全機制**：
- 所有交易都有完整的 PaymentLog 記錄
- 支援簽章驗證防止偽造回調

## 4. 安全性

### Q4.1：如何保護用戶資料和交易安全？

**答案**：

**多層安全防護**：

| 防護層 | 實作方式 |
|--------|----------|
| **認證** | JWT Token + Google OAuth |
| **密碼** | bcrypt 加密儲存 |
| **CSRF 防護** | Double Submit Cookie 模式 |
| **速率限制** | 多層級 + 敏感端點加強保護 |
| **HTTP 安全標頭** | Helmet 配置（HSTS、X-Frame-Options 等）|
| **CSP** | Content Security Policy 防 XSS |
| **依賴掃描** | pnpm audit + CI 自動掃描 |
| **生產保護** | Swagger 文件自動隱藏 |

**密碼重設流程**：
1. 用戶請求重設
2. 系統發送 Email（使用 Resend 服務）
3. Token 有效期限管理
4. 一次性使用後失效

---

### Q4.2：API 有做存取控制嗎？

**答案**：

**角色權限系統**：

| 角色 | 權限說明 |
|------|----------|
| `USER` | 一般會員：下單、報名活動、管理個人資料 |
| `VIP` | VIP 會員：專屬折扣、優先報名 |
| `STAFF` | 員工：查看訂單、管理行程 |
| `ADMIN` | 管理員：完整後台權限 |

**存取控制實作**：
```typescript
// 使用 Guard 保護端點
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles('ADMIN')
async createProduct() { ... }
```

**公開 vs 保護端點**：
- 公開：產品列表、產品詳情、健康檢查
- 保護：下單、個人資料、管理功能
