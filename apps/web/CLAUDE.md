## App Router 規範

### Client vs Server Components

**預設是 Server Component**，只在需要時加 `'use client'`：

```tsx
// ✅ Server Component（預設）- 可以直接 fetch 資料
export default async function ProductsPage() {
  const products = await fetchProducts()
  return <ProductList products={products} />
}

// ✅ Client Component - 使用 hooks 時必須標記
'use client'

import { useState } from 'react'

export function AddToCartButton() {
  const [isLoading, setIsLoading] = useState(false)
  return <button>加入購物車</button>
}
```

**何時需要 `'use client'`**：
- 使用 React Hooks (`useState`, `useEffect`, `useCallback`...)
- 使用瀏覽器 API (`window`, `localStorage`)
- 使用事件處理器 (`onClick`, `onChange`)
- 使用第三方客戶端函式庫

### 動態路由參數

Next.js 15+ 的 `params` 是 Promise，需要用 `use()` 解開：

```tsx
'use client'

import { use } from 'react'

interface PageProps {
  params: Promise<{ id: string }>
}

export default function ProductPage({ params }: PageProps) {
  const { id } = use(params)  // 解開 Promise
  // ...
}
```

### 導航

```tsx
// ✅ 使用 next/link
import Link from 'next/link'
<Link href="/products">產品</Link>

// ✅ 程式化導航
import { useRouter } from 'next/navigation'
const router = useRouter()
router.push('/products')
router.back()

// ✅ 取得路徑和參數
import { usePathname, useSearchParams } from 'next/navigation'
const pathname = usePathname()
const searchParams = useSearchParams()
```

### useSearchParams 需要 Suspense

使用 `useSearchParams()` 的元件必須包在 Suspense 中：

```tsx
'use client'

import { Suspense } from 'react'
import { useSearchParams } from 'next/navigation'

function SearchContent() {
  const searchParams = useSearchParams()
  const query = searchParams.get('q')
  return <div>搜尋: {query}</div>
}

export default function SearchPage() {
  return (
    <Suspense fallback={<div>載入中...</div>}>
      <SearchContent />
    </Suspense>
  )
}
```

### 路由保護

使用 `ProtectedRoute` 元件：

```tsx
// app/dashboard/page.tsx
'use client'

import { ProtectedRoute } from '@/components/ProtectedRoute'
import { DashboardContent } from './DashboardContent'

export default function DashboardPage() {
  return (
    <ProtectedRoute>
      <DashboardContent />
    </ProtectedRoute>
  )
}
```

## 狀態管理

### Zustand Store（與之前相同）

```typescript
// stores/authStore.ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      user: null,
      token: null,
      isAuthenticated: false,
      setAuth: (user, token) => set({ user, token, isAuthenticated: true }),
      logout: () => set({ user: null, token: null, isAuthenticated: false }),
    }),
    { name: 'auth-storage' }
  )
)
```

## API 整合

### 環境變數

Next.js 使用 `process.env.NEXT_PUBLIC_*`：

```typescript
// ✅ 正確：Next.js 環境變數
const API_URL = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3001'

// ❌ 錯誤：Vite 語法
const API_URL = import.meta.env.VITE_API_URL  // 不支援
```

### .env 檔案

```env
# .env.local
NEXT_PUBLIC_API_URL=http://localhost:3001
NEXT_PUBLIC_USE_MOCK=false
```
