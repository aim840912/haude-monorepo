# Test Runner

執行專案測試並分析結果，提供修復建議。

## 使用方式

```
/test-runner [app名稱] [測試類型]
```

- `app名稱`：web | admin | api | all（預設 all）
- `測試類型`：unit | e2e | all（預設 all）

## 執行步驟

### 1. 檢查測試環境

```bash
# 確認依賴已安裝
pnpm install
```

### 2. 執行測試

**API 測試（NestJS）**：
```bash
cd apps/api && pnpm test
```

**Web 測試（如果有）**：
```bash
cd apps/web && pnpm test
```

**Admin 測試（如果有）**：
```bash
cd apps/admin && pnpm test
```

### 3. 分析測試結果

對於每個失敗的測試：
1. 識別失敗原因
2. 定位相關程式碼
3. 提供修復建議

### 4. 輸出報告

## 測試狀態圖例

| 狀態 | 說明 |
|------|------|
| PASS | 測試通過 |
| FAIL | 測試失敗 |
| SKIP | 測試被跳過 |

## 輸出格式

```markdown
# 測試報告

## 摘要
- 總測試數：[N]
- 通過：[N]
- 失敗：[N]
- 跳過：[N]

## 失敗測試分析

### [測試名稱]
- **檔案**：[path/to/test.spec.ts]
- **錯誤**：[錯誤訊息]
- **原因**：[分析原因]
- **修復建議**：[具體建議]

## 下一步
- [ ] [修復項目 1]
- [ ] [修復項目 2]
```

## 常見測試問題

| 問題 | 可能原因 | 解決方案 |
|------|----------|----------|
| 資料庫連接失敗 | 環境變數未設定 | 檢查 .env 檔案 |
| 型別錯誤 | 介面不匹配 | 更新型別定義 |
| Timeout | 非同步操作過慢 | 增加 timeout 或優化 |
