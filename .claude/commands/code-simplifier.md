# Code Simplifier

檢視最近的變更並簡化程式碼，遵循 YAGNI 原則。

## 執行步驟

### 1. 識別變更範圍

```bash
git diff HEAD~1 --name-only
```

### 2. 針對每個變更的檔案，檢查以下項目

**移除冗餘**：
- [ ] 未使用的 imports
- [ ] 未使用的變數和函數
- [ ] 重複的程式碼邏輯
- [ ] 過度的 null checks（可用 optional chaining 簡化）

**提升可讀性**：
- [ ] 過長的函數（建議 < 30 行）
- [ ] 過深的巢狀（建議 < 3 層）
- [ ] 命名是否清晰表達意圖
- [ ] 是否有更簡潔的寫法

**遵循專案慣例**：
- [ ] 是否符合 CLAUDE.md 中的錯誤記錄
- [ ] 是否遵循現有的程式碼風格

### 3. 執行簡化

對每個需要簡化的檔案：
1. 說明問題所在
2. 提供簡化後的程式碼
3. 解釋為什麼這樣更好

### 4. 驗證

```bash
pnpm type-check
```

## 簡化原則

| 原則 | 說明 |
|------|------|
| YAGNI | 不要為「可能」的需求寫程式碼 |
| DRY | 相同邏輯出現 3+ 次才考慮抽取 |
| KISS | 選擇最簡單可行的解決方案 |

## 輸出格式

```markdown
# 簡化報告

## 檔案：[path/to/file.tsx]

### 問題 1：[問題描述]
- 原因：[為什麼這是問題]
- 修改：[具體變更]

### 問題 2：...

## 驗證結果
- 型別檢查：[通過/失敗]
```
